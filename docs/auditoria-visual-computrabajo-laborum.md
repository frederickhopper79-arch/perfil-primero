# Auditoria visual comparativa - Computrabajo/Laborum

## Hallazgos utiles para Perfil Primero

1. Las bolsas grandes reducen friccion inicial con busqueda directa, categorias claras y rutas rapidas por cargo/region.
2. El usuario entiende mejor una pagina cuando ve ejemplos concretos de cargos, sueldo, modalidad y estado.
3. Las secciones largas pierden fuerza si mezclan landing, directorio, beneficios, seguridad y filtros en una sola bajada.
4. La pagina debe separar dos intenciones: postulante que crea perfil y empresa que busca talento.
5. Los filtros pertenecen al contexto de busqueda empresarial, no al hero principal para postulantes.

## Cambios ya aplicados

- Hero enfocado en trabajador/postulante.
- CTA primario unico: crear perfil.
- CTA empresa secundario.
- Eliminacion de porcentaje hardcodeado.
- Storyboard con caso Juan.
- Filtros/directorio movidos a `/empresa`.
- Beneficios consolidados.
- Lenguaje simple, sin jerga startup.

## Recomendaciones siguientes

1. Crear una pagina `/empleos` publica con ofertas visibles, si se decide mostrar vacantes sin romper la metodologia invertida.
2. Mejorar cards de candidatos dentro de `/empresa` con comparacion lado a lado mas densa.
3. Agregar buscador sticky dentro del panel empresa, no en home.
4. Implementar estados visuales de proceso mas compactos.
5. Probar con usuarios reales si entienden la diferencia entre publicar perfil anonimo y postular a oferta.
